# AGENTS.md
채용 브랜딩 커리어 사이트 MVP – Antigravity 에이전트 운영 규칙

> 이 문서는 본 프로젝트에서 **모든 에이전트가 반드시 따라야 하는 최상위 기준(헌법)**입니다.  
> Antigravity에서 실행되는 모든 미션은 본 문서를 **항상 컨텍스트로 참조**해야 합니다.

---

## 1. 프로젝트 개요

### 프로젝트 목적
- 토스(Toss) 스타일의 채용 브랜딩 기반 커리어 사이트 MVP 구현
- 채용 공고와 채용 브랜딩 콘텐츠(Crew Story, Inside Us)를 하나의 탐색 흐름으로 연결
- 후보자가 헤매지 않고 **빠르게 판단 → 지원**까지 도달하도록 UX 설계

### MVP의 정의
- “예쁜 커리어 페이지”가 아니라  
  **채용 전환을 고려해 설계된 제품(Product)**

---

## 2. MVP 범위 (Must Have)

### 포함 범위
- Home
- Job Hub (직무 허브)
- Job Detail (채용 공고 상세)
- Content Detail (Crew Story / Inside Us)
- About (회사 소개)
- 통합 검색 (공고 + 콘텐츠)

### 제외 범위 (MVP 아님)
- 로그인/회원가입
- 관리자 대시보드
- 복잡한 권한 관리
- ATS 직접 연동 (링크 수준까지만 허용)

---

## 3. UX 핵심 원칙 (최우선)

### 3.1 Toss 스타일 UX 원칙
1. **한 섹션 = 하나의 메시지**
2. 긴 설명 ❌ → 짧은 문장 ⭕
3. 카드 중심 UI
4. 여백을 통한 정보 구분
5. 즉각적인 행동 유도 CTA

### 3.2 탐색 경험 원칙
- 직무 기준 탐색이 최우선
- 공고 ↔ 콘텐츠 이동은 항상 **1클릭**
- 사용자가 “다음에 무엇을 해야 할지” 항상 명확해야 함

---

## 4. CTA 규칙 (중요)

### 허용되는 CTA (행동형)
- 지원하기
- 이 역할 자세히 보기
- 지금 채용 중인 직무 보기
- 이 팀 채용 공고 보기
- 팀 이야기 더 보기

### 금지되는 CTA
- 더 알아보기
- 자세히
- 확인하기
- Learn more (의미 불명확)

---

## 5. 정보 구조(IA) 규칙

### 허용
- 카드형 리스트
- 섹션 단위 흐름
- Sticky 요약 영역 (Job Detail)

### 금지
- 탭 과다 사용
- 한 페이지에 모든 정보 나열
- 사이드바 기반 복잡한 네비게이션
- 스크롤을 강요하는 긴 설명

---

## 6. 페이지별 필수 조건

### Home
- 회사 한 줄 소개가 최상단
- 직무 선택 CTA가 가장 빠른 행동
- 10초 안에 “볼 가치” 판단 가능

### Job Hub
- 직무 설명 → 공고 → 사람 이야기 → 일하는 방식 순서
- 직무와 무관한 콘텐츠 노출 금지

### Job Detail
- 지원 CTA 항상 화면에 노출
- 역할/문제 중심 설명 우선

### Content Detail
- 콘텐츠는 전환의 끝이 아님
- 반드시 관련 채용 공고 CTA 포함

---

## 7. 기술 구현 가이드 (MVP 기준)

- Framework: Next.js (App Router)
- Styling: Tailwind CSS
- Data: 정적 JSON → 이후 CMS/DB 확장 가능
- SEO 고려 (페이지 단위 메타)

---

## 8. 산출물 규칙 (모든 미션 공통)

모든 미션 완료 후, 에이전트는 반드시 아래를 제공해야 함:

1. 생성/수정된 파일 목록
2. 주요 컴포넌트 설명
3. UX 의사결정 요약
4. AGENTS.md 기준 준수 여부 체크

---

## 9. 실패 기준 (이 중 하나라도 해당 시 재작업)

- 토스 스타일이 아닌 설명 위주 UI
- CTA가 추상적이거나 행동을 유도하지 않음
- 공고와 콘텐츠가 분리되어 탐색 흐름이 끊김
- “왜 이렇게 설계했는지” 설명 불가

---
